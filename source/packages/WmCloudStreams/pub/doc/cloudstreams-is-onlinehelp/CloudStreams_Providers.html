Providers
A provider is a logical grouping of connectors for a SaaS vendor and is installed on Integration Server as an Integration Server package.
A connector represents an integration interface to the provider using a specific communication channel.
You can create one or more connections for a connector at design time to use in integrations. A cloud connection is required for configuring and running a cloud connector service. You must create and enable a cloud connection before you can create cloud connector services. See the respective provider documents for connection configuration information and how to configure connections for a connector. Connection configurations vary for each connector.
A cloud connector service defines an interaction that the connector will perform on the SaaS back end. A cloud connector service has input and output signatures and is based on the interaction definitions defined in the connector and contains appropriate logic for executing the interaction and getting the response from the back end. You can create a cloud connector service using a wizard in Software AG Designer. Before configuring a cloud connector service, you must assign it a connection that you created earlier.
Connectors support connectivity with streaming APIs and processing of streaming API events. Streaming API access is available as a connector capability and you can integrate with a streaming API using a connector. You can use CloudStreams connections for connecting to streaming APIs. You can create a CloudStreams connector listener, select a subscription channel from a list of available channels for an endpoint, and configure the action(s) to be applied on the incoming events. Additionally, you can configure transport or communication related parameters for a CloudStreams connector listener as well as enable and disable a CloudStreams connector listener. A CloudStreams listener receives the streaming API events and processes the received events. You create connector listeners using the Software AG Designer. Before you create a connector listener, ensure that the CloudStreams connector associated with your cloud application provider is installed. Also ensure that a cloud connection pool is created for that connector. For details, see the Administering webMethods CloudStreams document.
To open the Providers options screen, click Solutions > CloudStreams > Providers in the navigation panel.
Click on a provider and from the Connectors screen, you can:
Enable, disable, and delete CloudStreams connectors.
Create, enable, disable, and delete connections.
Enable, disable, and configure listener connections. Also, replay lost Salesforce events by selecting the appropriate replay option.
Note:
For details, see the webMethods Service Development Help, Administering webMethods CloudStreams document, and the documentation specific to your CloudStreams provider.
